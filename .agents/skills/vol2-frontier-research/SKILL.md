# Vol II Frontier Research

Frontier work is allowed here, but it must still be audit-grade.

## Default posture

- Start from the strongest established live surface, not from inherited ambition.
- Prefer a smaller theorem with a closed proof to a grand conjectural package presented as solved.
- Separate:
  - proved core
  - conditional bridge
  - conjectural extension
  - heuristic physical picture

## Research loop

1. Identify the exact frontier claim or question.
2. Inventory proved inputs already available in the repo.
3. Identify the first missing lemma or obstruction, not just the final desired theorem.
4. Stress-test the proposal with the hostile examiners:
   - Beilinson
   - Witten
   - Costello
   - Gaiotto
   - Drinfeld
   - Kontsevich
5. Produce the strongest truthful formulation:
   - theorem
   - proposition
   - conditional theorem
   - conjecture
   - research direction
6. If executable consequences exist, wire them into compute/tests.

## Swarm rule

- Do not spawn sub-agents unless the user explicitly asks for delegation, sub-agents, or parallel agent work.
- If the user does authorize a swarm, split by independent axes with disjoint write scopes and a clear integration plan.
